About this tutorial

This tutorial teaches data manipulation for GIS using agricultural production data (2025).

Each exercise teaches data manipulation using a syntax (the R tidyverse syntax) that is the direct equivalent of a SQL command you may already know:

SQL tidyverse verb What it does
WHERE filter() Keep rows matching a condition
SELECT col1, col2 select() Keep only certain columns
ORDER BY col DESC arrange(desc()) Sort rows
GROUP BY + SUM() group_by() + summarise() Aggregate by group
JOIN … ON left_join() Combine two tables

Operations are chained with the pipe %>% — read it as “then”:

agriculture %>%           # start with this table, THEN
  filter(crop == "Rice")  # keep only Rice rows

Exercise 1 · filter()

The concept

filter() keeps only the rows that match your conditions — exactly like WHERE in SQL.

SQL

SELECT * FROM agriculture
WHERE crop = ‘Maize’
  AND prod_kg > 5000;

tidyverse

agriculture %>%
  filter(crop == “Maize”,
         prod_kg > 5000)

Key differences from SQL:

  • Use == (double equals) to test equality — = alone is assignment in R
  • Separate multiple conditions with a comma (means AND)
  • Text values use "double quotes"

Exercise

From the agriculture table, keep only rows where crop is "Maize" and prod_kg is greater than 5000.

  filter(___ == "___",
         ___ > ___)
# The first condition filters on the crop column
agriculture %>%
  filter(crop == "Maize",
         ___ > ___)
# The second condition filters on the prod_kg column
agriculture %>%
  filter(crop == "Maize",
         prod_kg > ___)
agriculture %>%
  filter(crop == "Maize",
         prod_kg > 5000)

Exercise 2 · select()

The concept

select() keeps only the columns you name — like listing column names after SELECT in SQL.

SQL

SELECT district_id, crop, value_real
FROM agriculture;

tidyverse

agriculture %>%
  select(district_id, crop, value_real)

Key points:

  • Column names are written without quotes
  • Order in select() determines column order in the output
  • Use -col to drop a column instead of keeping it

Exercise

From the agriculture table, keep only the columns district_id, crop, and value_real.

agriculture %>%
  select(___, ___, ___)
# Name the three columns separated by commas — no quotes needed
agriculture %>%
  select(district_id, ___, ___)
agriculture %>%
  select(district_id, crop, value_real)

Exercise 3 · arrange()

The concept

arrange() sorts rows — like ORDER BY in SQL. Wrap a column name in desc() to sort descending.

SQL

SELECT * FROM agriculture
ORDER BY value_real DESC;

tidyverse

agriculture %>%
  arrange(desc(value_real))

Key points:

  • Default is ascending — arrange(col) = ORDER BY col ASC
  • Descending requires wrapping: arrange(desc(col)) = ORDER BY col DESC
  • Sort by multiple columns: arrange(crop, desc(value_real))

Exercise

Sort the agriculture table by value_real from highest to lowest.

agriculture %>%
  arrange(___(___))
# Use desc() inside arrange() for descending order
agriculture %>%
  arrange(desc(___))
agriculture %>%
  arrange(desc(value_real))

Exercise 4 · group_by() + summarise()

The concept

group_by() defines groups, then summarise() computes one value per group — equivalent to GROUP BY + aggregate functions in SQL.

SQL

SELECT district_id,
       SUM(prod_kg) AS total_prod_kg
FROM agriculture
GROUP BY district_id;

tidyverse

agriculture %>%
  group_by(district_id) %>%
  summarise(total_prod_kg =
    sum(prod_kg, na.rm = TRUE))

Key points:

  • Always use na.rm = TRUE inside sum() — otherwise a single NA makes the whole sum NA
  • Common aggregates: sum(), mean(), n() (count rows), n_distinct(), max(), min()
  • group_by() alone changes nothing visible — it only takes effect in the next summarise() or mutate()

Exercise

For each district_id, calculate the total production (sum of prod_kg). Name the result column total_prod_kg.

agriculture %>%
  group_by(___) %>%
  summarise(total_prod_kg = ___(prod_kg, na.rm = TRUE))
# group_by() takes the column you want to group on
agriculture %>%
  group_by(district_id) %>%
  summarise(total_prod_kg = ___(prod_kg, na.rm = TRUE))
# Use sum() to add up all values within each group
agriculture %>%
  group_by(district_id) %>%
  summarise(total_prod_kg = sum(prod_kg, na.rm = TRUE))
agriculture %>%
  group_by(district_id) %>%
  summarise(total_prod_kg = sum(prod_kg, na.rm = TRUE))

Exercise 5 · left_join()

The concept

left_join() adds columns from a second table by matching on a shared key — equivalent to JOIN … ON in SQL.

SQL

SELECT d.district, a.crop,
       a.prod_kg, d.population
FROM agriculture a
JOIN districts d
  ON a.district_id = d.district_id;

tidyverse

agriculture %>%
  left_join(districts,
    by = “district_id”) %>%
  select(district, crop,
           prod_kg, population)

Key points:

  • left_join() keeps all rows from the left table and adds matching columns from the right
  • The by = argument names the shared key column — must match exactly in both tables
  • Chain select() afterwards to keep only the columns you need

Exercise

Join agriculture and districts using district_id as the key. Then keep only the columns district, crop, prod_kg, and population.

agriculture %>%
  left_join(___, by = "___") %>%
  select(___, ___, ___, ___)
# The second table to join is districts; the key is district_id
agriculture %>%
  left_join(districts, by = "district_id") %>%
  select(___, ___, ___, ___)
# Select the four columns: district (from districts), crop, prod_kg, population
agriculture %>%
  left_join(districts, by = "district_id") %>%
  select(district, crop, prod_kg, population)

  left_join(districts, by = "district_id") %>%

Bonus · Combining verbs

Real analysis chains multiple verbs

Run each chunk to see how verbs combine. Feel free to modify them!

Top 10 Bean-producing districts by name:

agriculture %>%
  filter(crop == "Bean") %>%
  group_by(district_id) %>%
  summarise(total_prod_kg = sum(prod_kg, na.rm = TRUE)) %>%
  arrange(desc(total_prod_kg)) %>%
  left_join(districts, by = "district_id") %>%
  select(district, total_prod_kg) %>%
  head(10)

Total production and value per crop:

agriculture %>%
  group_by(crop) %>%
  summarise(
    total_prod_kg = sum(prod_kg,    na.rm = TRUE),
    total_value   = sum(value_real, na.rm = TRUE),
    n_districts   = n_distinct(district_id)
  ) %>%
  arrange(desc(total_value))

Bar chart — total production by crop:

agriculture %>%
  group_by(crop) %>%
  summarise(total_prod_kg = sum(prod_kg, na.rm = TRUE)) %>%
  ggplot(aes(x = reorder(crop, total_prod_kg),
             y = total_prod_kg,
             fill = crop)) +
  geom_col(width = 0.6) +
  coord_flip() +
  scale_fill_manual(values = c(Bean = "#a78bfa", Maize = "#f5a623", Rice = "#3ecf8e")) +
  scale_y_continuous(labels = scales::comma) +
  labs(title = "Total production by crop 2025",
       x = NULL, y = "Production (kg)") +
  theme_minimal(base_size = 13) +
  theme(legend.position = "none")

Summary · SQL → tidyverse cheatsheet

SQL tidyverse Notes
SELECT col1, col2 select(col1, col2) No quotes on column names
SELECT * (default — omit select) All columns by default
SELECT DISTINCT col distinct(col) Removes duplicate rows
WHERE col = ‘x’ filter(col == ‘x’) == for equality
WHERE col > 5 AND col2 = ‘y’ filter(col > 5, col2 == ‘y’) Comma means AND
WHERE col IN (‘a’,‘b’) filter(col %in% c(‘a’,‘b’)) Use %in% for lists
WHERE col IS NULL filter(is.na(col)) NA not NULL in R
ORDER BY col ASC arrange(col) Ascending is default
ORDER BY col DESC arrange(desc(col)) Wrap in desc()
GROUP BY col group_by(col) Pair with summarise()
SUM(col) AS total summarise(total = sum(col, na.rm=TRUE)) na.rm drops NAs
COUNT(*) AS n summarise(n = n()) n() counts rows in group
COUNT(DISTINCT col) summarise(n = n_distinct(col)) Unique values per group
JOIN t2 ON t1.k = t2.k left_join(t2, by = ‘k’) Keeps all left rows
SELECT col, col*2 AS double mutate(double = col * 2) mutate() adds new columns
