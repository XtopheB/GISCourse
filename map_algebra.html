Local operation
Add two agricultural yield layers
A local operation computes a new value for each cell using only the values at that same position across input layers. No neighbourhood is involved — each cell is processed independently.

You have two yield raster layers (kg/ha) collected in different surveys for the same region. Compute the combined yield Layer C = A + B for each cell. Fill in Layer C.
Layer C[i,j] = Layer A[i,j] + Layer B[i,j]
Top-left cell: A = 3, B = 5 → C = 8. Work row by row, position by position.

03
Focal operation
Smooth the production raster with a 3×3 mean filter
A focal operation computes each output cell value using a neighbourhood of surrounding cells. A 3×3 mean filter is commonly applied to agricultural production rasters to smooth out local noise and produce the Focal_production raster.

Apply the filter: for each cell, compute the mean of all cells in its 3×3 window. Edge cells use only available neighbours. Fill in all output cells. Round to 1 decimal place.
Focal_production[i,j] = mean( all cells in 3×3 window centred on [i,j] )
-- edge cells: use only the neighbours that exist (4 for corners, 6 for edges)
Centre cell (row 2, col 2): mean of all 9 values. Top-left corner: only 4 cells available — mean those 4. Round to 1 decimal.
04
Zonal statistics
Compute total production per district
Zonal statistics summarise a value raster within zones defined by a separate raster. Here, zones represent districts and values represent agricultural production (kg).

The District grid assigns each cell to a district (1, 2, or 3). The Production grid contains production values. For each district, compute the total production (sum of all cell values in that district). Fill in the summary table.
Total production[d] = SUM( Production[i,j] where District[i,j] = d )
District 1 cells are at positions (0,0), (0,2), (1,1) — find their production values and add them up. Do the same for districts 2 and 3.
05
Threshold masking
Mask out low-production cells
Masking removes cells that do not meet a condition by setting them to NoData. In agriculture, this is used to exclude non-productive land, water bodies, or urban pixels from analysis.

Apply a production threshold: cells where production < 5 → NoData. All other cells keep their original production value. Write ND for NoData cells.
if production < 5 → NoData (cell excluded from analysis)
else → keep original production value
Scan each cell. If value < 5, write ND. Otherwise copy the value as-is. Check every position — even large values should be copied unchanged.

Weighted overlay
Compute an agricultural suitability score
Weighted overlay combines multiple criteria layers into a single suitability score. Each layer is multiplied by its weight, and results are summed. Weights must sum to 1.0.

Combine three agricultural factors — Soil fertility, Rainfall, and Market access — to produce a suitability score for each cell. Adjust the weights, compute the output, and fill in your answers. Round to 1 decimal.
Suitability[i,j] = w1 × Soil[i,j] + w2 × Rainfall[i,j] + w3 × Market[i,j]
-- constraint: w1 + w2 + w3 = 1.0
Top-left cell: Soil=6, Rainfall=4, Market=7. Multiply each by its weight and sum. With equal weights (0.33): 6×0.33 + 4×0.33 + 7×0.33 ≈ 5.6.
07
Change detection
Detect changes in agricultural production between two years
Change detection subtracts a before-raster from an after-raster to identify increases, decreases, or stability in a variable over time.

Grid Y2000 shows production (kg/ha) in 2000, and Y2010 shows the same in 2010. Compute the difference (Y2010 − Y2000), then classify each cell: G = Gain (diff > 1), L = Loss (diff < −1), S = Stable (−1 to +1).
Difference[i,j] = Y2010[i,j] − Y2000[i,j]
if diff > 1 → G (Gain)
if diff < −1 → L (Loss)
else → S (Stable)
G — Gain (>+1)
L — Loss (<−1)
S — Stable
Top-left: Y2000=3, Y2010=7 → diff = 4 → G (Gain). Remember: Stable means the absolute difference is ≤ 1.
08
Local operation — division
Compute agricultural production per capita
After applying a focal smoothing filter to the production raster (Focal_production), we divide by the Population raster to obtain production per capita — how many kg of food are produced per person in each cell.

Divide each cell of the Focal_production raster by the corresponding Population cell. Round your answers to 1 decimal place.
Units: Focal_production in kg · Population in persons → result in kg / person
Prod_per_capita[i,j] = Focal_production[i,j] ÷ Population[i,j]
Top-left: Focal_production = 6.0, Population = 3 → 6.0 / 3 = 2.0. Round all results to 1 decimal place.
09
Normalization & derived index
Normalize production per capita and compute vulnerability
Normalization rescales values to a 0–1 range so that rasters with different units can be compared. Here we normalize the production per capita raster (2000), then derive a Vulnerability index: the less food produced per person, the more vulnerable the cell.

Step 1: Normalize each cell value using min-max scaling.
Step 2: Compute vulnerability as 1 minus the normalized value. Fill in both output grids.
The minimum and maximum values of the input raster are shown above the grid.
Normalized[i,j] = ( value − min ) ÷ ( max − min )
Vulnerability[i,j] = 1 − Normalized[i,j]
Find min and max from the input grid. For each cell: normalized = (v − min) / (max − min). Then vulnerability = 1 − normalized. Round to 2 decimal places.
10
Masking + local multiplication
Compute production for agricultural cells (2000)
This exercise chains two operations. First, a yield raster (kg/ha) is masked by an agriculture binary raster (1 = agricultural cell, 0 = non-agricultural). Non-agricultural cells become zero. Then the masked yield is multiplied by the cell area (ha) to obtain total production per cell.

Step 1: Multiply Yield × Agriculture mask → Masked yield (0 for non-agri cells).
Step 2: Multiply Masked yield × Cell area (2 ha per cell) → Production (kg).
Assumption: within each district, all agricultural cells share the same yield value (uniform yield). In reality, yields vary by soil type, irrigation, and farming practices.
Masked_yield[i,j] = Yield[i,j] × Agri_mask[i,j]
Production[i,j] = Masked_yield[i,j] × Cell_area (= 2 ha)
Step 1: if agri mask = 0, masked yield = 0 regardless of yield value. Step 2: multiply masked yield by 2 (cell area). Non-agri cells produce 0 kg.
