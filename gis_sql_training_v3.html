Explore the two tables
These are the two tables available in all exercises. The districts table holds district names and population. The agriculture table holds crop production data — linked by district_id.
01 · SQL
Filter rows with WHERE
Write a query that returns all rows from the agriculture table where the crop is 'Maize' and the prod_kg is greater than 5000.
agriculture
district_idtext
yeartext
croptext
area_hainteger

Use WHERE with AND to combine two conditions. Text values need quotes: crop = 'Maize'
SELECT * FROM agriculture WHERE crop = 'Maize' AND prod_kg > 5000;

Aggregate data with GROUP BY
For each district_id, calculate the total production (sum of prod_kg). Name the result column total_prod_kg.
agriculture
district_idtext
croptext
prod_kginteger
value_realdecimal
Use SUM(prod_kg) AS total_prod_kg to aggregate. Then GROUP BY district_id.
SELECT district_id, SUM(prod_kg) AS total_prod_kg FROM agriculture GROUP BY district_id;

Combine tables with JOIN
Join agriculture and districts to get the district name, crop, prod_kg, and population for all rows. Use district_id as the join key.

Use JOIN … ON to link both tables: FROM agriculture a JOIN districts d ON a.district_id = d.district_id. Then select using aliases: d.district, a.crop, etc.
SELECT d.district, a.crop, a.prod_kg, d.population FROM agriculture a JOIN districts d ON a.district_id = d.district_id;
